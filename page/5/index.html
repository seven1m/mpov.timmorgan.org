ShrinkSafe with Prototype 1.6

Aug 21, 2008
I didn’t find this information anywhere, so here it is for the Google archives… To get Prototype to run after squeezing it through ShrinkSafe, you need to put back every instance of the variable $super. ShrinkSafe replaces variable names with shortened versions, and because of some Prototype voodoo, it breaks on this one name. There are about 8 places you’ll find it, and unfortunately, it takes some manual hunting once ShrinkSafe has changed a bunch of names, but there you have it.

Ditch Pagination

Jul 2, 2008
I build web apps. Almost every one has a screen or two somewhere that shows a listing, and at the bottom it has pagination links, e.g. next page, page 3, etc. For a recent app I’ve been working on, the interface is very search-driven, i.e. the user enters some criteria the results are displayed. As an experiment, I left off the pagination links. In this app, there is no “next page.” Instead, if the results are not on the first page, then the user must refine their search.

Git For People Who Know Svn, Part 2

Mar 3, 2008
For Part 2 of our 60-second Git tutorial, we’ll look at some super simple branching and merging. git branch tells you which branch you’re on. git branch -a lists all branches, even remote ones. git checkout mybranch checks out the branch you want. git checkout -b newbranch creates a new branch from the one you’re on, and checks it out for you. git checkout master && git merge mybranch merges changes from one branch to your master branch. If you don’t yet want to commit the changes, you can do git merge --no-commit mybranch instead.
